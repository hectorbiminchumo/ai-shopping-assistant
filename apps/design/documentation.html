FORMA Design System

Design
Language

A minimal, high-performance design language for FORMA's digital presence. Monochrome palette, grotesque type, precise spacing — everything scales from 320px to 1440px+.

View store Product page

Principles

Three values guide every decision in this system.

01 — Clarity
No decoration
Every element earns its place. High contrast, generous whitespace, legible type at all sizes.
02 — Motion
Purposeful transitions
The single easing curve cubic-bezier(.22,.61,.36,1) makes every interaction feel responsive, not performative.
03 — Adaptability
Light & dark, all sizes
Dark mode is first-class. Every token has a dark counterpart. Fluid type with clamp() throughout.

Image Slot

A custom element (<image-slot>) that provides user-fillable image placeholders throughout the design. Users drop image files directly onto slots, and the image persists via a sidecar JSON file. Supports drag-and-drop, file browse, image replacement, removal, and a reframe/crop mode (double-click).

Attributes
AttributeValuesDefaultDescription
idstring—Required for persistence. Without it, the image is session-only.
shaperect / rounded / circle / pillroundedPreset border-radius on the frame
radiusnumber (px)12Corner radius when shape="rounded"
maskCSS clip-path—Arbitrary clip-path override. Overrides shape.
fitcover / contain / fillcoverObject-fit. Double-click reframe only works with cover.
positionCSS object-position50% 50%For fit=contain/fill
placeholderstring"Drop an image"Empty-state caption
srcURL—Fallback image URL. A user drop overrides it; clearing shows src again.
Persistence

Dropped images are saved to .image-slots.state.json as data URLs (WebP, max 1200px on longest side). The sidecar is read on page load and written on every image change. When running inside the omelette runtime (window.omelette.writeFile), writes go through the host bridge. Outside it, the component is read-only.

Accepted formats: PNG, JPEG, WebP, AVIF. SVG and GIF are rejected for security and single-frame limitations respectively.

Sidecar: .image-slots.state.json · Max dimension: 1200px · Format: WebP q=0.85
Reframe mode (double-click)

When fit="cover", double-clicking a filled slot enters reframe mode. The full image spills past the mask with a translucent ghost and corner drag handles. Drag to reposition, corner-drag to scale (aspect-locked), wheel to zoom toward cursor. Press Escape or click outside to commit.

Reframe stores {s, x, y} alongside the image URL for persistent crops across responsive resizes
Live demo
Drag any PNG/JPEG/WebP image onto these slots to test. Auto-persists if writable.

Accessibility

Design decisions and patterns that make the interface usable by everyone.

Practices

Reduced motion@media(prefers-reduced-motion:reduce) disables notch hint animation. All transitions use --ease and respect OS settings.
Keyboard navigationAll interactive elements are buttons or links · Escape closes panels · Enter activates · visible :focus-visible outlines
ARIA labelsaria-label on all icon-only buttons (Search, Cart, Account, Close, Prev, Next) · aria-label="Search assistant" on search panel

Form labelsAll inputs have associated labels or aria-label · newsletter form has explicit name + required
